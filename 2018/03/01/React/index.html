. React의 장점

  1. Javascript 기반 : 동의. 그런데 이건 Vue도 마찬가지.
  2. Component기반 : 글쎄? 꼭 React가 아니더라도 워낙 커스터마이즈드 된 컴포턴트가 많아서 정말 재사용할 수 있을지 의문.
  3. 단방향 데이터 흐름 : 글쎄? 양방향이 더 좋은 것 같기도 한데. 일관성이 필요할까? 더 공부해봐야.
  4. 활성화된 생태계 : 동의. 어느 기술이든 활성화된 생태계는 정말 필요한 듯..

. React는 프레임워크가 아니라 UI라이브러리라고.

. create-react-app : 모듈번들러인 webpack을 사용하지 않고도, react 작업을 가능하게 스캐폴딩 해주는 툴.

. 모듈번들러 : 리액트 코드, 최신 자바스크맄트 등을 브라우저가 이해할 수 있는 코드로 변경.

리액트는 프레임워크가 아니라 UI라이브러리라고.

프레임워크과 라이브러리의 차이
Ruby on rails Django.

Wehpack : 모듈번들러. 리액트 코드 또는 최신 자바스크립트를 브라우저가 이해할 수 있는 코드로 변경

Create-react-app 웹펙과 같은 툴을 사용하지 않고, 리액트 작업 가능하게.
