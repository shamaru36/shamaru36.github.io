Pro Git

Git이나 GitHub라는 용어에 대해서는 많이 들어봤는데 역시 직접 해보는 것만큼 선명하게 이해하는 방법은 없는 것 같다. Git은 SVN처럼 형상관리를 도와주는 하나의 툴을, GitHub는 인터넷(?) 저장소를 의미한다.

  1. Git 다운로드 및 설치
    . 두 번째 옵션이 Default값이지만 이대로 진행하면 cmd를 실행했을 때 Git 버전의 cmd가 실행되도록 변경됨

!Hexo 설치만이 목적이라면 굳이 이하의 작업을 할 필요는 없다.

  1. Git bash 실행

  2. Repository로 사용할 폴더 생성 후 해당 폴더로 이동

  3. Repository에 수정한 내역을 일괄적으로 Git과 연동

  4. Git에 의해 추적되는 파일 커밋

  5. 커밋 메세지 작성(vim 편집기)
    I : 편집 모드 시작
    Esc : 편집 모드 종료
    : : 명령 모드 시작
    (:)w : 저장
    (:)q : 명령 모드 종료

다양한 옵션을 익혀두어야 할 것 같다.
