Markdown


Markdown


Markdown is a lightweight and easy-to-use syntax for styling all forms of writing on the GitHub platform.

Markdown is a way to style text on the web. You control the display of the document; formatting words as bold or italic, adding images, and creating lists are just a few of the things we can do with Markdown.
Mostly, Markdown is just regular text with a few non-alphabetic characters thrown in, like # or *.

- https://guides.github.com/features/mastering-markdown/


흔히 접하는 README.md 의 .md가 바로 마크다운 파일이었다. 문서파일의 일종으로, 디자인을 위한 몇몇 문법이 정해져 있다. 여기서
문법이 정해져 있다는 건, 특정 기호를 HTML 태그 형식으로 바꿔준다는 것을 의미한다. 각각의 태그에 어떤 스타일을 적용할 지는 별개의 문제이므로 마크다운 에디터나 Hexo theme의 영향을 받는다. (결국 스타일 처리가 되어있지 않다면 인용문구나 코드블럭이라 하더라도 텍스트처럼 보일 수 있다)


  1. Headers : # (#의 개수가 HN 태그의 N값이 된다)

  2. Emphasis : * 또는 _ (italic), * 또는 __ (bold), ** 또는 ___ (italic & bold)

  3. Lists
    3-1. Unordered : * 또는 + 또는 -
    3-2. Ordered : No.

  4. Images : !Alt Text

  5. Links : [링크](url)

  6. Blockquotes : >

  7. Inline code : `(code)`

  8. Code Block : ```(language) ~ ``` 또는 ~(language) ~ ~
