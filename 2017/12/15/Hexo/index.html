결론적으로, 굉장히 간단하게 인터넷에 블로그를 생성할 수 있게 해준다.
GitHub를 제외하고는 특정 사이트에 의존할 필요도 없고 도메인을 구입할 필요도 없다.

사전에 Git과 Node.js(npm)가 설치되어 있어야 한다. 이 두 가지가 제대로(!) 설치만 되어있다면 별도로 Git이나 Node.js와 관련된 설정을 할 필요는 없다. 여기서 ‘제대로’라는 건, 시스템 속성의 환경 변수에 이 두 가지가 등록되어야 함을 의미한다. 이 점을 잘 알지 못해 며칠을 고생했다. 특히 Git의 경우 exe 파일을 실행할 때 옵션 선택에 주의해야 한다.

두 번째 옵션이 default로 선택되어 있는데 그대로 설치했다가 cmd를 실행할 때 window cmd 대신 git cmd가 실행되길래 다시 설치할 때 제대로 읽어보지 않고 첫 번째 옵션으로 변경했었다…. 설명을 잘 보면, ‘This is the safest choice as your PATH will not be modified at all‘이라고 적혀있다.

사전 준비가 완료되면 cmd 창을 열고 이하의 명령어를 실행하면 된다.

$ npm install hexo-cli -g
$ hexo init “폴더명”
$ cd “폴더명”
$ npm install

참고로, Git이 환경 변수로 잡혀있지 않다면 ‘hexo init’ 명령어를 실행했을 때
Clone에 실패하여 Copy한다는 메세지를 보게 되는데 이러면 안되는 것 같다.
여기서 넘어가더라도, 나중에 배포하는 명령어를 실행했을 때 정상적으로 작동하지 않는다.

문제없이 스캐폴딩이 잘 되었다면 이하의 명령어를 통해 로컬 서버를 띄워볼 수 있다. (Vue.js 볼 때만 해도 스캐폴딩이 무슨 개념인지 정확히 몰랐었는데..)
명령어 실행 후 http://localhost:4000 에 접속해 보면 된다.

$ hexo server

블로그를 커스터마이징하려면 루트 폴더의 _config.yml을 수정하면 되고,
글을 등록하기 위해서는 이하의 명령어를 통해 root 디렉토리의 source/_posts 폴더 아래 마크다운 파일을 생성한 다음 마크다운 문법에 맞게
내용을 작성하면 된다.

$hexo new [layout] “타이틀”

레이아웃에는 post(source/_posts), page(source), draft(source/_draft)가 있는데
post가 default값으로, 생략 가능하다.(_config.yml의 default_layout 옵션)
. page : 해당 경로로 접근해야 볼 수 있는 페이지
. draft : 바로 게시하지 않고 작성하는 초안. 먼저 초안을 작성하고 완성된 뒤에 publish 명령어로 배포

제목에 띄어쓰기를 하려면 ‘-‘를 넣어주고,
한글이 깨지지 않게 하려면 마크다운 파일의 인코딩을 UTF-8로 지정해주어야 한다.

웹에 배포하기 위해서는 이하의 명령어를 실행하면 되는데, Git을 통해 배포하려면
‘$npm install hexo-deploy-git –save’를 먼저 실행하고 ‘_config.yml’에도
‘type : git, repo : 배포하고자 하는 URL, branch: master’를 추가로 등록해줘야 한다.

$ hexo generate
$ hexo deploy
($ hexo deploy -generate(=hexo d -g))
($ hexo generate –watch)

혹시 위의 명령어를 실행했는데도 반영이 제대로 되지 않는다면 이하의 명령어를 통해 publish 폴더를
초기화하고 다시 실행하면 된다.

$hexo clean

블로그의 테마를 변경하고 싶다면 이하의 경로에서 원하는 디자인의 코드를 받아서 themes 폴더에 넣고
_config.yml에서 layout 값을 바꿔주면 된다. 참고로, 이 블로그에 적용된 theme는 ‘edinburgh’이며
각 Post의 대표 이미지는 source 폴더에 새 폴더를 추가하여 적용하고 있다.

https://hexo.io/themes/index.html
